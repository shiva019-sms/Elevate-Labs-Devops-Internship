What is Docker?

Containerization Platform

Docker is an open-source platform that enables developers to package applications and their dependencies into containers for consistent deployment across environments.

Fast & Efficient

Containers are lightweight and start in seconds, making development and deployment faster than traditional virtual machines.

Isolated Environments

Each container runs in isolation, ensuring that applications don't interfere with each other and maintaining security.

Docker vs Virtual Machines

Docker Containers

  • ✓ Lightweight (MBs)
  • ✓ Boots in seconds
  • ✓ Shares host OS kernel
  • ✓ Less resource intensive

Virtual Machines

  • • Heavy (GBs)
  • • Boots in minutes
  • • Full OS required
  • • More resource intensive

Installation Guide

Install Docker on Linux (Ubuntu/Debian)

1

Update Package Index

sudo apt update
sudo apt upgrade -y
2

Install Prerequisites

sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
3

Add Docker's Official GPG Key

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
4

Add Docker Repository

sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
5

Install Docker Engine

sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io -y
6

Verify Installation

docker --version
docker info
7

Add User to Docker Group (Optional)

sudo usermod -aG docker $USER
newgrp docker

Install Docker on Windows

1

System Requirements

  • Windows 10 64-bit: Pro, Enterprise, or Education
  • Hyper-V and Containers Windows features enabled
  • BIOS-level hardware virtualization support
2

Download Docker Desktop

Visit Docker Desktop for Windows and download the installer.

3

Run the Installer

Double-click Docker Desktop Installer.exe and follow the installation wizard.

4

Start Docker Desktop

Launch Docker Desktop from the Start menu and wait for it to start.

5

Verify Installation

docker --version
docker run hello-world

Install Docker on macOS

1

System Requirements

  • macOS 10.15 or newer
  • Apple silicon (M1/M2) or Intel processor
  • At least 4GB RAM
2

Download Docker Desktop

Visit Docker Desktop for Mac and download the appropriate version for your chip (Intel or Apple Silicon).

3

Install Docker Desktop

Drag Docker.app to your Applications folder and launch it.

4

Verify Installation

docker --version
docker run hello-world

Essential Docker Commands

Image Commands

docker images

List all Docker images on your system

docker pull <image>

Download an image from Docker Hub

docker build -t <name> .

Build an image from a Dockerfile

docker rmi <image>

Remove an image

docker tag <source> <target>

Tag an image with a new name

Container Commands

docker run <image>

Create and start a container from an image

docker ps

List running containers

docker ps -a

List all containers (including stopped)

docker stop <container>

Stop a running container

docker start <container>

Start a stopped container

docker rm <container>

Remove a stopped container

docker exec -it <container> bash

Execute a command in a running container

Monitoring Commands

docker logs <container>

View container logs

docker logs -f <container>

Follow container logs in real-time

docker inspect <container>

Display detailed container information

docker stats

Display resource usage statistics

Cleanup Commands

docker system prune

Remove unused data

docker container prune

Remove all stopped containers

docker image prune

Remove unused images

docker volume prune

Remove unused volumes

Common Docker Run Flags

-d

Run container in detached mode (background)

-p

Publish container port to host (-p 8080:80)

-v

Bind mount a volume (-v /host:/container)

--name

Assign a name to the container

-e

Set environment variables (-e VAR=value)

-it

Interactive mode with TTY

--rm

Automatically remove container when it exits

--network

Connect container to a network

Understanding Dockerfile

A Dockerfile is a text document containing instructions to build a Docker image. Each instruction creates a layer in the final image.

Dockerfile Structure & Instructions

FROM

Specifies the base image

FROM python:3.9-slim

WORKDIR

Sets the working directory

WORKDIR /app

COPY

Copies files from host to container

COPY . /app

ADD

Similar to COPY but with URL support

ADD file.tar.gz /app

RUN

Executes commands during build

RUN pip install -r requirements.txt

CMD

Default command when container starts

CMD ["python", "app.py"]

ENTRYPOINT

Configures container as executable

ENTRYPOINT ["python"]

EXPOSE

Documents which ports to expose

EXPOSE 8080

ENV

Sets environment variables

ENV APP_ENV=production

VOLUME

Creates a mount point

VOLUME ["/data"]

USER

Sets the user for RUN, CMD, ENTRYPOINT

USER appuser

ARG

Defines build-time variables

ARG VERSION=1.0

Dockerfile Best Practices

Use Official Base Images

Start with official, verified images from Docker Hub

Minimize Layers

Combine related commands to reduce image size

Use .dockerignore

Exclude unnecessary files from build context

Order Instructions Wisely

Put frequently changing instructions at the end

Use Multi-Stage Builds

Reduce final image size by using multiple FROM statements

Run as Non-Root User

Improve security by avoiding root user

Practical Examples

Python Flask Application

app.py

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "<h1>Hello from Docker!</h1>"

@app.route('/health')
def health():
    return {"status": "healthy"}

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

requirements.txt

Flask==2.3.0
gunicorn==20.1.0

Dockerfile

# Use official Python runtime
FROM python:3.9-slim

# Set working directory
WORKDIR /app

# Copy requirements file
COPY requirements.txt .

# Install dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY app.py .

# Expose port
EXPOSE 5000

# Run the application
CMD ["python", "app.py"]

Build & Run Commands

# Build the image
docker build -t flask-app:v1 .

# Run the container
docker run -d -p 5000:5000 --name my-flask-app flask-app:v1

# View logs
docker logs my-flask-app

# Test the application
curl http://localhost:5000

Node.js Express Application

app.js

const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('<h1>Hello from Node.js Docker Container!</h1>');
});

app.get('/api/status', (req, res) => {
  res.json({ status: 'running', timestamp: new Date() });
});

app.listen(port, '0.0.0.0', () => {
  console.log(`App listening on port ${port}`);
});

package.json

{
  "name": "docker-node-app",
  "version": "1.0.0",
  "main": "app.js",
  "dependencies": {
    "express": "^4.18.2"
  },
  "scripts": {
    "start": "node app.js"
  }
}

Dockerfile

# Use official Node.js runtime
FROM node:18-alpine

# Set working directory
WORKDIR /usr/src/app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm install --production

# Copy application code
COPY app.js .

# Expose port
EXPOSE 3000

# Run the application
CMD ["npm", "start"]

Build & Run Commands

# Build the image
docker build -t node-app:v1 .

# Run the container
docker run -d -p 3000:3000 --name my-node-app node-app:v1

# View logs
docker logs -f my-node-app

# Test the application
curl http://localhost:3000

<!DOCTYPE html>

Dockerfile

# Use official Nginx image
FROM nginx:alpine

# Copy static files to Nginx directory
COPY index.html /usr/share/nginx/html/

# Expose port 80
EXPOSE 80

# Nginx starts automatically

Build & Run Commands

# Build the image
docker build -t nginx-site:v1 .

# Run the container
docker run -d -p 8080:80 --name my-nginx nginx-site:v1

# Test the application
curl http://localhost:8080

# Stop and remove
docker stop my-nginx
docker rm my-nginx

Multi-Stage Build (Go Application)

main.go

package main

import (
    "fmt"
    "net/http"
)

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "<h1>Hello from Go in Docker!</h1>")
}

func main() {
    http.HandleFunc("/", handler)
    fmt.Println("Server starting on :8080")
    http.ListenAndServe(":8080", nil)
}

Dockerfile (Multi-Stage)

# Stage 1: Build
FROM golang:1.20-alpine AS builder

WORKDIR /app

COPY main.go .

RUN go build -o main .

# Stage 2: Production
FROM alpine:latest

WORKDIR /root/

# Copy only the binary from builder
COPY --from=builder /app/main .

EXPOSE 8080

CMD ["./main"]

Build & Run Commands

# Build the image
docker build -t go-app:v1 .

# Run the container
docker run -d -p 8080:8080 --name my-go-app go-app:v1

# Check image size (notice how small it is!)
docker images go-app:v1

# Test the application
curl http://localhost:8080

Why Multi-Stage Builds?

  • Significantly reduces final image size
  • Keeps build tools out of production image
  • Improves security by minimizing attack surface
  • Faster deployment and startup times
