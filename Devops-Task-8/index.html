Step-by-Step Guide

Lets build your first CI/CD pipeline from scratch.

1

Create a GitHub Repository

Go to GitHub and create a new public repository named my-first-pipeline. Initialize it with a README.

2

Add a Simple Project

Create a simple index.html file in your repository. This will be the "project" we are checking.

<!DOCTYPE html> <html> <body> <h1>Hello World</h1> </body> </html>
3

Create Workflow Folder

In your repository, create the following directory structure. This is where GitHub looks for workflows.

.github/workflows/
4

Write Your First CI Pipeline

Create a file named ci.yml inside the workflows folder and add this content:

name: Simple CI Pipeline on: [push] jobs: build-test: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v3 - name: Run a script run: echo "Building and testing..."
5

Push Changes & Run Pipeline

Commit your changes and push them to GitHub. This will automatically trigger the workflow.

git add . git commit -m "Add CI pipeline" git push origin main
6

View Execution Logs

Go to the Actions tab in your GitHub repository. Click on the latest run to see the "build-test" job executing.

7

Environment Variables & Secrets

Store sensitive data in "Settings > Secrets". You can access them in YAML like this:

env: API_KEY: ${{ secrets.MY_API_KEY }}
8

Break the Pipeline Intentionally ❌

Edit your workflow to add a command that fails, to see how errors look.

- name: Cause Failure run: exit 1
9

Fix the Pipeline ✅

Remove the failing command and push again. Watch the status turn green!
